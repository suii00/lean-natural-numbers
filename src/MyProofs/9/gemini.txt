もちろん、`admit`（`sorry`の同義語）を削減するためのお手伝いをします。提示されたLeanコードは、楕円曲線の理論を厳密に証明しようとする素晴らしい試みですね。

`admit`を解消するには、主に2つのアプローチがあります。

1.  **計算による証明**: `ring`や`field_simp`のようなタクティクを使い、複雑な代数計算をLeanに実行させる。
2.  **理論に基づく証明**: 数学の定理（Mathlibに存在する、あるいは自明なもの）を適用して論理的に証明する。

一部の`admit`は、代数幾何学の非常に高度な定理に依存しており、これらをゼロから証明するのは現実的ではありません。その場合は、「構造的に受け入れる」という現在の方針が適切です。

以下に、各`admit`を解消するための具体的なコードと解説を示します。

### \#\# `AlgebraicAutomation`—代数計算の自動化

ここでの`admit`は、楕円曲線の加法公式が正しいことを代数的に示すものです。これは複雑ですが、Leanのタクティクで証明可能です。

#### **`addition_formula_complete` の修正案**

この証明は、分母を払い（`field_simp`）、多項式の等式として解く（`ring`）のが定石です。

```lean
lemma addition_formula_complete (E : EllipticCurve ℚ)
    (x₁ y₁ x₂ y₂ : ℚ)
    (h₁ : y₁^2 = x₁^3 + E.a * x₁ + E.b)
    (h₂ : y₂^2 = x₂^3 + E.a * x₂ + E.b)
    (hne : x₁ ≠ x₂) :
    let slope := (y₂ - y₁) / (x₂ - x₁)
    let x₃ := slope^2 - x₁ - x₂
    let y₃ := slope * (x₁ - x₃) - y₁
    y₃^2 = x₃^3 + E.a * x₃ + E.b := by
  -- 変数定義を展開
  dsimp only
  -- x₁ ≠ x₂ を利用して分母を払う
  field_simp [hne]
  -- 仮定 h₁, h₂ を等式変形に使えるようにする
  rw [show (y₂ - y₁)^2 = (x₂^3 - x₁^3) + E.a * (x₂ - x₁) by { rw [←h₂, ←h₁]; ring }]
  rw [show x₂^3 - x₁^3 = (x₂ - x₁) * (x₂^2 + x₂*x₁ + x₁^2) by ring]
  -- 再度分母を払う
  field_simp [hne]
  -- 残りの式は多項式の等式なのでringで証明完了
  ring
```

#### **`tangent_addition_formula` の修正案**

こちらも同様に、`field_simp`と`ring`を組み合わせて証明します。

```lean
lemma tangent_addition_formula (E : EllipticCurve ℚ)
    (x y : ℚ) (h : y^2 = x^3 + E.a * x + E.b) (hy : y ≠ 0) :
    let slope := (3 * x^2 + E.a) / (2 * y)
    let x₃ := slope^2 - 2 * x
    let y₃ := slope * (x - x₃) - y
    y₃^2 = x₃^3 + E.a * x₃ + E.b := by
  dsimp only
  -- y ≠ 0 より 2 * y ≠ 0
  have hy2_ne_zero : 2 * y ≠ 0 := mul_ne_zero (by norm_num) hy
  -- 分母を払う
  field_simp [hy2_ne_zero]
  -- y^2 を仮定hで置き換える
  rw [h]
  -- 多項式の等式をringで証明
  ring
```

**解説** 💡
[cite\_start]これらの証明が完成すれば、`EllipticCurveComplete.enhanced_add_points`内の2つの`admit` [cite: 14, 16] は、今証明した`addition_formula_complete`と`tangent_addition_formula`を`apply`することで解消できます。

### \#\# `IsogenyTheoryComplete`—同種写像理論

このセクションの`admit`は、高度な数学的定理に関するものです。

#### **`degree_one_isomorphism_complete`内の`admit`**

この定理には複数の`admit`が含まれていますが、1つは基本的な集合論の議論で証明できます。

  * [cite\_start]**`kernel_card_eq_degree` [cite: 23]**: **`admit`を維持推奨**
    **理由**: 「核の位数 = 次数」は同種写像の基本定理ですが、その証明は非常に高度です。Mathlibにこの定理がなければ、これを「公理」として受け入れるのが現実的です。

  * [cite\_start]**`φ.kernel = {Point.infinity}`の証明内の`admit` [cite: 25]**: **証明可能**
    **理由**: 「位数が1で、特定の元`e`を含む集合は、`{e}`という単元集合である」という事実は、集合論の基本から証明できます。

    ```lean
    have : φ.kernel = {Point.infinity} := by
      -- 「集合Sが元eをただ一つだけ含む」ことと「S = {e}」が同値であることを利用
      apply Set.eq_singleton_iff_unique_mem.mpr
      constructor
      · -- Point.infinity が核に含まれることは既に証明済み
        exact inf_in_kernel
      · -- 核に含まれる任意の元 y が Point.infinity であることを示す
        intro y hy_in_kernel
        -- もし y が infinity でないと仮定すると矛盾が生じる
        by_cases h_eq : y = Point.infinity
        · exact h_eq
        · -- {infinity, y} は核の部分集合
          have h_sub : {Point.infinity, y} ⊆ φ.kernel := by
            intro p hp; rcases (Set.mem_insert_iff.mp hp) with rfl | rfl;
            exact inf_in_kernel; exact hy_in_kernel
          -- 部分集合の位数は元の集合の位数以下
          have ncard_le := Set.ncard_le_ncard h_sub (Set.toFinite _) φ.kernel_finite
          -- {infinity, y} の位数は2
          have ncard_s_eq_2 : Set.ncard {Point.infinity, y} = 2 := Set.ncard_pair h_eq
          -- 2 ≤ 1 となり矛盾
          rw [ncard_s_eq_2, card_one] at ncard_le
          linarith
    ```

  * [cite\_start]**逆写像の存在と性質に関する`admit` [cite: 27]**: **`admit`を維持推奨**
    **理由**: これらも代数幾何学の深い結果（Véluの公式など）に依存するため、「構造的に受け入れる」のが最善のアプローチです。

### \#\# `EllipticCurveComplete.enhanced_eq_original`

  * [cite\_start]**`enhanced_eq_original`の`admit` [cite: 17]**: **証明可能（ただし注意点あり）**
    **理由**: もし`enhanced_add_points`と`add_points`の定義が本当に「完全一致」しているなら、以下の`rfl`（reflexivity）タクティクで証明が完了するはずです。

    ```lean
    theorem enhanced_eq_original (E : EllipticCurve ℚ) (P Q : Point ℚ E) :
        enhanced_add_points E P Q = add_points E P Q := by
      unfold enhanced_add_points
      -- add_pointsの定義も展開する必要がある。もしadd_pointsが
      -- 非公開(private)などで展開できない場合、この証明は困難になる。
      -- 最も堅牢な方法は、PとQのケースごと（infinityかaffineか）に証明すること。
      cases P <;> cases Q <;> rfl
    ```

    **注意**: `rfl`で通らない場合、2つの関数の実装に微妙な違いがあるか、Leanが定義的等価性を自動で認識できていない可能性があります。その場合は、`simp`や`rw`を使って手動で式を揃える必要があります。
