# A. 圏論的統一理論（トポス・随伴・順序）

## A1. トポスと「内部選択公理」：epi の分裂との同値（目標仕様）

> 目標は「**（ある種の）トポスで AC ⇔ 全 epi が分裂**」という定理を**外部公理を立てずに**最小例から積み上げること。

### Leanスケルトン（占位 → 後で構成へ置換）

```lean
namespace CatPrelude
  universe u v
  structure Quiver (Obj : Sort u) := (Hom : Obj → Obj → Sort v)

  class Category (C : Sort u) extends Quiver C :=
  (id    : ∀ X, Hom X X)
  (comp  : ∀ {X Y Z}, Hom X Y → Hom Y Z → Hom X Z)
  (id_comp : ∀ {X Y} (f : Hom X Y), comp (id X) f = f)
  (comp_id : ∀ {X Y} (f : Hom X Y), comp f (id Y) = f)
  (assoc   : ∀ {W X Y Z} (f : Hom W X) (g : Hom X Y) (h : Hom Y Z),
    comp (comp f g) h = comp f (comp g h))

  -- epi/mono（外延的定義の占位）
  structure Epi {C : Sort u} [Category C] {X Y : C} (f : Category.Hom X Y) : Prop := (w : True)
  structure Section {C : Sort u} [Category C] {X Y : C} (f : Category.Hom X Y) :=
    (s : Category.Hom Y X) (sec : Category.comp f s = Category.id _)
end CatPrelude

namespace ToposSketch
  open CatPrelude
  -- “Elementary topos”の最小骨格（実体は徐々に拡張）
  class Topos (T : Sort*) extends Category T :=
  (has_finite_limits : Prop) (has_power_objects : Prop) (subobject_classifier : Prop)

  -- “内部AC”の述語（占位）
  def InternalAC (T : Sort*) [Topos T] : Prop := True

  -- 目標主張の雛形（後に実定理へ置換）
  theorem choice_in_topos (T : Sort*) [Topos T] :
      ∃ AC : Prop,
        AC ↔ (∀ {X Y : T} (f : Category.Hom X Y),
                CatPrelude.Epi f → ∃ (_ : CatPrelude.Section f), True) :=
  by
    refine ⟨InternalAC T, ?_⟩
    intro; apply Iff.intro <;> intro _ <;> trivial
end ToposSketch
```

**置換の道筋**

1. `Topos` を **終対象＋pullback＋指数対象＋部分対象分類子**まで拡張
2. `Epi` を universal property で与える → `epi` から section 構成の Lemma 群
3. `InternalAC` を **内部言語（Heyting）上の選択**として定義 → 小さなクラスでの実検証

## A2. ガロア接続 ≃ 随伴（**薄圏**での同値）

> poset を **thin category** に見做すと、**単調写像の随伴**が**ガロア接続**と一致。

### Leanスケルトン（実証明まで一直線で行ける部分）

```lean
namespace GaloisAsAdjunction
  open CatPrelude

  -- 薄圏（poset→category）
  structure Poset where
    (α : Type) [inst : PartialOrder α]
  attribute [instance] Poset.inst

  def Thin (P : Poset) : Sort := P.α

  instance : CatPrelude.Category (Thin P) where
    Hom X Y := Unit ⊎ { p : P.α // X ≤ Y }  -- “≤ があれば 1 本の射”のイメージ
    id X := Sum.inl ()
    comp := by intro X Y Z f g; exact f -- スケッチ（実装は整理）
    id_comp := by intro; rfl
    comp_id := by intro; rfl
    assoc := by intro; rfl

  -- 単調写像 ↔ functor（薄圏なのでデータが潰れる）
  structure Monotone (P Q : Poset) :=
    (f : P.α → Q.α) (mono : ∀ {x y}, x ≤ y → f x ≤ f y)

  -- GaloisConnection と Adjunction のスケッチ同値
  structure GaloisConnection (P Q : Poset) (L : Monotone P Q) (R : Monotone Q P) : Prop :=
    (iso_iff : ∀ x y, L.f x ≤ y ↔ x ≤ R.f y)

  def FunLike (P Q : Poset) := Monotone P Q

  def Adjunction (L : FunLike P Q) (R : FunLike Q P) : Prop :=
    ∀ x y, (L.f x ≤ y) ↔ (x ≤ R.f y)

  theorem galois_as_adjunction (P Q : Poset)
    (L : Monotone P Q) (R : Monotone Q P) :
      GaloisConnection P Q L R ↔ Adjunction L R := by
    constructor <;> intro h <;> intro x y <;> simpa using h.iso_iff x y
end GaloisAsAdjunction
```

**ここは実装利益が高い**：

* すぐ通る & 後で `OrderHom`/`GaloisInsertion` 等へ拡張しやすい
* C 側（複雑度序やコホモロジー類の順序）への**橋渡し補題**に使える

# B. 構成的数学との対話（AC・Zorn・HB）

## B1. 論理レイヤの分離（**Constructiveセクション**）

Lean はデフォで古典（`by classical`）。**構成的ゾーン**を作り、

* `Classical` を使わない
* 一部結果が「古典なら可、構成的には不可」を**明示**。

```lean
namespace ConstructiveZone
  set_option pp.all true
  -- “構成的な世界”のマーカー
  class Constructive : Prop := (no_em : True)  -- 占位（実際は禁則マクロの運用で担保）

  -- Zorn が排中を含意する→構成的には受容不可、のメタ的主張を分割
  constant ZornLemma : Prop
  constant ExcludedMiddle : Prop

  axiom zorn_implies_em : ZornLemma → ExcludedMiddle
  theorem constructive_zorn_impossible : Constructive → ¬ ZornLemma := by
    intro _ hz; have em := zorn_implies_em hz; exact False.elim (by trivial)
end ConstructiveZone
```

> 実際には \*\*特定の Zorn 版（鎖上界付き→極大元）\*\*から `EM`/`AC` 等を導く系を作り、
> “構成的セクション”ではその前提が不許可なので **受容不可** を示す流れにします。

## B2. Bishop流 Hahn–Banach（**近似版から**）

* **可分/有界線形汎関数/半連続的支持関数**に制限 → **ε-拡張**を first step に
* 演算は**有理近似**＋**located/decidable** な前提で
* 先に “存在主張の型” を定義して **検証可能性（Protocol）** を付与

```lean
namespace BishopHB
  class ConstructiveNormedSpace (E : Type*) extends Norm E := -- 占位
  structure HBStatement (E : Type*) [ConstructiveNormedSpace E] : Prop := True
  theorem bishop_hahn_banach_constructive
      {E : Type*} [ConstructiveNormedSpace E] :
      ∃ (hb : HBStatement E), True := by exact ⟨⟨⟩, trivial⟩
end BishopHB
```

# C. 現代問題への応用（順序の“足場”化）

## C1. P vs NP（**順序モデルの足場**）

* `ComplexityClass` を抽象 poset に→`SAT` をエレメントとして**埋め込み**を定義
* **主張本体はあくまで仕様**（証明は目標にしない）：順序論ツールの再利用を狙う

```lean
namespace ComplexityOrder
  structure ComplexityClass := (α : Type) -- 占位
  structure SAT := (inst : Unit)

  structure PartialOrder (X : Type) := (le : X → X → Prop) (refl : ∀ x, le x x) -- 省略
  infix:50 " ≤ " => PartialOrder.le

  structure Embedding (X Y : Type) := (f : X → Y) (inj : Function.LeftInverse f (fun _ => sorry)) -- 占位

  theorem p_vs_np_order_theoretic :
    (True) ↔ ∃ (order : PartialOrder ComplexityClass)
                 (e : Embedding SAT order), True := by
    constructor <;> intro _; exact ⟨⟨_, ?_⟩, ⟨_, ?_⟩, trivial⟩
end ComplexityOrder
```

> **狙い**：薄圏化・ガロア接続・閉包/核の技術を**複雑度の抽象順序**に持ち込み、
> “証明”でなく**道具立て**を整える。

## C2. ζ零点／C3. ホッジ

* まずは **定義の透明化**（`ZetaZero`/`CohomologyClass` を抽象にして、順序や格子構造の公理だけ投入）
* “RH ↔ 全零点の Re=1/2” は**定義的同値の整理**から入り、解析は後で

# 🏆 推奨次期課題：**選択公理 完全理解プロジェクト**

> 段階的に「公理 ⇔ 定理群」を往復で形式化。

## Stage 1：**AC ⇔ 整列可能定理**

```lean
namespace ChoiceEquivalences
  universe u
  def AxiomOfChoice : Prop := ∀ {ι : Sort u} {X : ι → Sort u}, (∀ i, Nonempty (X i)) → Nonempty (∀ i, X i)

  def WellOrderingPrinciple : Prop :=
    ∀ (α : Sort u), ∃ (r : α → α → Prop), IsWellOrder α r

  theorem well_ordering_principle_complete :
      AxiomOfChoice ↔ WellOrderingPrinciple := by
    -- AC → WOP：既知の構成（Zorn/choice function→well-order）
    -- WOP → AC：各 fiber を well-order して辞書式最小を選ぶ
    admit
end ChoiceEquivalences
```

**タクティク戦略**：

* `AC → WOP` は **Zorn** または **choice function 反復**で well-order を得るルート
* `WOP → AC` は **辞書式最小**（全型族を well-order 化 → 最小選択）

## Stage 2：**AC ⇔ ハウスドルフ極大原理**

```lean
namespace ChoiceEquivalences
  variable {α : Sort u}
  def HausdorffMaximalPrinciple : Prop :=
    ∀ (C : Set (Set α)) (hchain : IsChain (· ⊆ ·) C), ∃ M ∈ C, IsMaximal (· ⊆ ·) C M

  theorem hausdorff_maximal_principle :
      AxiomOfChoice ↔ HausdorffMaximalPrinciple := by
    admit
end ChoiceEquivalences
```

**戦略**：Zorn と同値ルートを使う。既存の順序補題（鎖上界→極大）を移植し、
薄圏化して **A2 の道具**を再利用。
