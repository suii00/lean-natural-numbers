def Matrix.dotProductᵣ{α : Type u_1} [Mul α] [Add α] [Zero α] {m : ℕ} (a b : Fin m → α) :
α
dotProduct with better defeq for Fin

Equations
Matrix.dotProductᵣ a b = FinVec.sum (FinVec.seq (FinVec.map (fun (x1 x2 : α) => x1 * x2) a) b)
について調査してください


了解しました。調査結果を要点でまとめます。

概要: Matrix.dotProductᵣ は Fin m → α を添字にもつベクトル向けに、定義等価性（defeq）を良くするために再帰で定義したドット積です。Fin の「先頭＋残り」分解に沿って簡約でき、小さい次元の計算や simp 展開が素直になります。

本リポジトリでの定義位置:

src/MyProjects/LinearAlgebra/A1/zenla1_session04test.lean:256（名前空間 ZenLA1.Session04.FinMul.Matrix）
定義（抜粋）:
m = 0: dotProductᵣ a b = 0
m = m+1: dotProductᵣ a b = a 0 * b 0 + dotProductᵣ (fun i => a i.succ) (fun i => b i.succ)
何が「defeqに良い」のか:

ふつうの Matrix.dotProduct は ∑ i, a i * b i の形（Finset 合計）で、Fin に対する具体的な分解は補題（例: Fin.sum_univ_succ）を使って進めます。
dotProductᵣ は「先頭要素＋残り」による再帰定義なので、rfl/simp で直接
a 0 * b 0 + dotProductᵣ (tail a) (tail b) へ簡約されます。![…] など固定長ベクトルでも素直に展開されます。
「Equations … = FinVec.sum (FinVec.seq (FinVec.map …) …)」について:

ベクトルを FinVec n α := Fin n → α と見て
FinVec.map₂ f a b := fun i => f (a i) (b i)
FinVec.sum v := ∑ i, v i
FinVec.seq (fs : FinVec n (α → β)) (xs : FinVec n α) := fun i => fs i (xs i)
と書けば、（加法構造があれば）次の等価が成り立ちます:
dotProductᵣ a b = FinVec.sum (FinVec.map₂ (· * ·) a b)
かつ FinVec.map₂ g a b = FinVec.seq (FinVec.map g a) b
したがって表示の
FinVec.sum (FinVec.seq (FinVec.map (fun (x1 x2) => x1 * x2) a) b)
は上の等価を書き換えたものです。
標準の dotProduct との橋渡し（追加で入れておくと便利な補題）:

加法モノイド構造がある場合、Fin に対して次が（帰納法で）証明できます。
@[simp] lemma dotProductᵣ_zero … : dotProductᵣ (m:=0) a b = 0
@[simp] lemma dotProductᵣ_succ … : dotProductᵣ (m:=m+1) a b = a 0 * b 0 + dotProductᵣ (fun i => a i.succ) (fun i => b i.succ)
lemma dotProductᵣ_eq_sum (a b : Fin m → α) [AddCommMonoid α] [Mul α] : dotProductᵣ a b = ∑ i, a i * b i
これにより、必要に応じて Matrix.dotProduct と相互に書き換え可能です。
行列積との関係:

同ファイルの mulᵣ は
mulᵣ A B i j = dotProductᵣ (fun k => A i k) (fun k => B k j)
と定義されており、Fin 指標での展開が容易になります。
使い分けの指針:

小さい固定次元（Fin 2/3/4）の具体計算や等式検証には dotProductᵣ（や mulᵣ）が向きます。simp だけで「先頭＋残り」に分解でき、数式計算に落としやすいです。
一般添字・既存の行列論の補題（Matrix.dotProduct, mulVec まわり）を活用するときは従来の ∑ 形を使う方が流れが良いことがあります。上の橋渡し補題で随時切り替え可能です。
簡単な使用例:

example (a b c d : α) [Mul α] [Add α] [Zero α] : Matrix.dotProductᵣ ![a,b] ![c,d] = a*c + b*d := by simp [Matrix.dotProductᵣ]
example (A : Matrix (Fin 2) (Fin 2) α) (x : Fin 2 → α) [Mul α] [Add α] [Zero α] : (ZenLA1.Session04.FinMul.Matrix.mulᵣ A (Matrix.col x)) 0 0 = _ := by simp [ZenLA1.Session04.FinMul.Matrix.mulᵣ, Matrix.dotProductᵣ]
