BFS正当性の難易度について


結論：中～高難度です（このプロジェクト方針だと「高」に寄ります）。

理由（簡潔）：

BFSは“正しさの層”が3段ある
停止性（燃料・訪問集合が単調に増える）
到達性（見つかれば到達可能）
最短性（最初に見つかった距離が最小）
の 最短性 が一番重く、
レベル（depth）と frontier の不変条件を明示して証明する必要がある。
今の実装は List と Finset の混在・filter/flatMap で、
**「frontier が長さ d の到達点のみ」**を言語化する補題が必要。
このプロジェクトだと：

Day 1–2 の #eval は簡単
Day 3–4 の RankHomLe も補題1本で行ける
BFS正当性は“Week 5–6”の見積りは妥当（証明が長くなる）
おすすめ方針（Bourbaki精神に沿って小さく）

まず 非形式的な正当性（コメント）
次に 弱い補題だけ形式化
pathLength = some d → d ≤ n（燃料上界）
pathLength = some d → reachable（到達性のみ）
最短性は後回し（ここが本番）
必要なら、**「BFS正当性の最小セット」**として
どの補題から始めるかのロードマップを出します。

BFS正当性の難易度を考えるとVec2Qを含めどう進めるべきか


結論：BFS正当性は後回し、当面は「RankCore の実用例を増やす」方針が最も堅いです。
Vec2Q はそのまま主軸に据えてOKで、GraphDist は “計算例を出すだけ” の扱いに留めるのが安全です。

理由（短く）：

BFS最短性の形式化は 証明コストが高い（不変条件が必要）。
いまの優先度（実用性・Bourbaki的な構造）には、RankCore の実例拡充が一番効く。
Vec2Q は Closure_Basic.lean を活かせて 低コストで信頼性が高い。
