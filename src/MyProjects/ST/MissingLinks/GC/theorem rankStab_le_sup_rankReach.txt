theorem finite_stabilizes、theorem rankStab_le_sup_rankReach、どちらが難しいですか


難しいのは rankStab_le_sup_rankReach です。

理由（簡潔に）

finite_stabilizes は「有限集合上の単調・拡大的写像の反復がいつか止まる」という標準事実で、Finite と「増加列は有限で打ち止め」を使えば比較的素直に詰められます。
rankStab_le_sup_rankReach は WithTop の sInf/iSup を絡めた不等式で、rankReach の定義展開や iSup の比較・sInf の操作が必要になり、補助補題も多めです。
どちらから着手しますか？


rankStab_le_sup_rankReachの証明戦略


戦略案（Lean 実装の流れ）を箇条書きでまとめます。

全体の方針

finite_stabilizes を使って安定化回数の最小値を取り出し、その最小値で新しい元が出ることを示す。
その新しい元の rankReach がちょうどその最小値になり、iSup がそれ以上であることから結論。
具体ステップ

有限性から最小安定化回数を取り出す

h : ∃ n, C.StabilizesAt s₀ n を finite_stabilizes から得る。
n := Nat.find h とおき、
have hstab : C.StabilizesAt s₀ n := Nat.find_spec h
have hrank : C.rankStab s₀ = (n : WithTop ℕ) := by simp [rankStab, h]
Nat.find_min' h より、m < n なら ¬StabilizesAt s₀ m.
n=0 の場合

rankStab s₀ = 0 なら 0 ≤ ⨆ x, rankReach x s₀ は自明（bot_le 相当）で終わり。
n = k+1 の場合：新しい元の存在

¬StabilizesAt s₀ k と C.iter_subset_succ s₀ k より
C.iter k s₀ ⊂ C.iter (k+1) s₀ が従う。
よって x を
x ∈ C.iter (k+1) s₀ ∧ x ∉ C.iter k s₀
で取れる（by classical + by_contra で subset→eq を否定）。
この x の到達回数が k+1 であること

x ∈ C.iter (k+1) s₀ から ReachableIn x s₀ (k+1)。
単調性 iter_mono で、m ≤ k なら C.iter m s₀ ⊆ C.iter k s₀。
よって x ∉ C.iter m s₀、つまり ReachableIn x s₀ m は成り立たない。
したがって到達回数の最小値は k+1。
Leanでは補助補題を作るのが楽です：
rankReach_le_of_reachable（到達できれば rankReach ≤ n）
rankReach_eq_of_reachable_of_not_reachable_lt
（ReachableIn と m < n に対して非到達なら rankReach = n）
これで rankReach x s₀ = (k+1 : WithTop ℕ)。
iSup との比較

le_iSup で rankReach x s₀ ≤ ⨆ x, rankReach x s₀ を得て、
(k+1 : WithTop ℕ) ≤ ⨆ x, rankReach x s₀
hrank と合わせて結論。
必要になりそうな補助

Nat.find_min' を使うので simp [rankStab, h] の形を作る。
反証で「subset が成り立てば等号になる」を使って差分要素 x を得る。
rankReach の下限側は sInf の一般補題ではなく、「最小の到達回数」型の補題を自前で書くと整理しやすいです。
必要なら、この方針に沿った小補題（rankReach_le_of_reachable など）から順に実装案を出します。
