# 構造塔とモダリティ：グラフ到達可能性による統一的理解

## 概要

このドキュメントは、構造塔（Structure Tower）とモーダル論理（Modal Logic）の深い関係を、グラフの到達可能性という具体例を通じて解説します。

## なぜこの例が重要か？

### 1. 3つの理論の統合

構造塔理論は以下を統一的に扱います：

```
代数的構造     グラフ構造        論理的構造
（線形包）     （到達可能性）    （モダリティ）
    ↓              ↓                ↓
必要な生成元数  最短経路長      証明の深さ
    ↓              ↓                ↓
     ━━━━━━━━  minLayer  ━━━━━━━━
              （リソースの最小量）
```

### 2. 時間発展と段階付き情報

**モーダルオペレータ** □ₙ(v) の解釈：

| 分野 | □ₙ(v) の意味 | minLayer(v) の意味 |
|-----|-------------|-------------------|
| グラフ理論 | v が n ステップで到達可能 | v への最短距離 |
| 時相論理 | v が n 時刻までに成立 | v が初めて成立する時刻 |
| 認識論理 | v が n 段階で証明可能 | v の証明に必要な最小深さ |
| 計算理論 | v が n ステップで計算可能 | v の計算の最小複雑度 |

## 実装の3段階アプローチ

### Version 1: 簡約版（概念理解）

**目的**: 構造塔の形を理解する

```lean
def reachableIn (...) : Prop := True  -- 常に到達可能
def shortestDistance (...) : ℕ := 0    -- 距離は常に0
```

**利点**:
- ✅ すべての証明が自明（`trivial`）
- ✅ 公理の役割が明確
- ✅ 型レベルでの理解に集中できる

**層の構造**:
```
layer(0) = 全頂点  ← すべてが即座に到達可能
layer(1) = 全頂点
layer(2) = 全頂点
```

### Version 2: 半具体版（直感を養う）

**目的**: 実際の階層構造を体験する

```lean
def reachableIn (k : ℕ) (v : Fin n) : Prop := v.val ≤ k
def shortestDistance (v : Fin n) : ℕ := v.val
```

**利点**:
- ✅ 手で計算できる
- ✅ 層が実際に異なる
- ✅ 視覚化が可能

**線形パスの例**: 0 → 1 → 2 → 3 → 4

```
時刻 k=0:  layer(0) = {0}              □₀ が真なのは頂点0のみ
時刻 k=1:  layer(1) = {0, 1}           □₁ が真なのは頂点0,1
時刻 k=2:  layer(2) = {0, 1, 2}        □₂ が真なのは頂点0,1,2
時刻 k=3:  layer(3) = {0, 1, 2, 3}     □₃ が真なのは頂点0,1,2,3
時刻 k=4:  layer(4) = {0, 1, 2, 3, 4}  □₄ で全体を覆う
```

**minLayerの意味**:
```lean
minLayer(0) = 0  -- 根は0時刻で観測
minLayer(1) = 1  -- 頂点1は1時刻で初めて観測
minLayer(2) = 2  -- 頂点2は2時刻で初めて観測
minLayer(3) = 3  -- 頂点3は3時刻で初めて観測
minLayer(4) = 4  -- 頂点4は4時刻で初めて観測
```

### Version 3: 完全版（一般理論）

**目的**: 任意のグラフに適用

```lean
def reachableIn : ℕ → Fin n → Prop
  | 0, v => v = root           -- 0ステップでは根のみ
  | k+1, v => reachableIn k v  -- 既に到達済み
              ∨ ∃u, reachableIn k u ∧ hasEdge u v  -- または1ステップで拡張

noncomputable def shortestDistance := Nat.find (...)  -- 実際の最短経路
```

**利点**:
- ✅ 任意のグラフ構造に対応
- ✅ BFS/DFSアルゴリズムの形式化

**課題**:
- ⚠️ 証明が複雑（有限性の議論）
- ⚠️ 計算不可能（noncomputable）

## モダリティ理論との対応

### Kripke意味論としての解釈

グラフ G = (V, E) は**Kripkeフレーム**と見なせる：

- **世界**: 頂点 v ∈ V
- **アクセス関係**: 辺 (u, v) ∈ E
- **真理値**: 命題 p が世界 v で真

構造塔の層は、**時間的モダリティ**の意味論を与える：

```
□ₖ p := "p は k ステップ以内の世界で真"
◇ₖ p := "p は k ステップ以内のある世界で真"
```

### モダリティの公理との対応

| モダリティ公理 | 構造塔での解釈 | Lean での証明 |
|-------------|--------------|--------------|
| □ₙ p ⇒ □ₙ₊₁ p | 単調性 | `monotone` |
| p ⇒ ◇₀ p | 被覆性 | `covering` |
| □ₙ p ∧ □ₙ q ⇒ □ₙ (p ∧ q) | 交差保存 | `layer` の定義 |

### 証明論との対応

構造塔は**証明の深さ**を形式化：

```
layer(0) = 公理から直接従う命題
layer(1) = 1回の推論規則で証明可能
layer(2) = 2回の推論規則で証明可能
...
layer(n) = n回の推論規則で証明可能
```

**minLayer(φ)** = 命題 φ の証明に必要な最小ステップ数

## 具体例：線形パスでの計算

### 設定
```
グラフ: 0 → 1 → 2 → 3 → 4
根: 0
```

### 各時刻での観測可能な頂点

```lean
-- 時刻0: 根のみ観測
example : modalBox 0 0 = True   ✓
example : modalBox 0 1 = False  ✗
example : modalBox 0 2 = False  ✗

-- 時刻1: 根と隣接頂点
example : modalBox 1 0 = True   ✓
example : modalBox 1 1 = True   ✓
example : modalBox 1 2 = False  ✗

-- 時刻2: 2ステップ到達可能
example : modalBox 2 0 = True   ✓
example : modalBox 2 1 = True   ✓
example : modalBox 2 2 = True   ✓
example : modalBox 2 3 = False  ✗
```

### モダリティの性質

**単調性**: □ₙ ⇒ □ₙ₊₁
```lean
lemma modalBox_mono (k : ℕ) (v : Fin 5) :
    modalBox k v → modalBox (k + 1) v
```

**最小性**: minLayer は初めて真になる時刻
```lean
lemma minLayer_characterization (v : Fin 5) :
    modalBox (minLayer v) v
    ∧ ∀ k < minLayer v, ¬modalBox k v
```

## 他の数学構造への応用

### 1. 代数的構造（既存実装）

**線形包**: span({v₁, ..., vₙ})

```
layer(0) = {0}           -- 零ベクトル
layer(1) = span({e₁, e₂}) -- 1つの基底で生成
layer(2) = ℚ²           -- 全空間
minLayer(v) = 必要な基底ベクトルの最小個数
```

### 2. 位相的構造（既存実装）

**位相的閉包**: A̅ = A ∪ A'

```
layer(0) = A            -- 元の集合
layer(1) = A ∪ A'       -- 極限点を追加
layer(2) = (A ∪ A')'    -- 極限点の極限点
minLayer(x) = x が初めて閉じる段階
```

### 3. グラフ構造（本実装）

**到達可能性**: reachable(v, k)

```
layer(k) = k ステップで到達可能な頂点
minLayer(v) = v への最短距離
```

### 4. 確率論的構造（フィルトレーション）

**停止時間**: τ : Ω → ℕ

```
layer(k) = ℱₖ（k 時刻までの可測集合）
minLayer(ω) = τ(ω)（ω で決定される時刻）
```

## 統一的視点：「段階付き情報」

すべての例は、**段階付き情報の開示**という統一的な解釈を持つ：

| 例 | 情報 | 段階 | minLayer の意味 |
|---|-----|-----|---------------|
| 線形包 | ベクトル | 基底の個数 | 表現に必要な最小基底数 |
| 位相閉包 | 点 | 閉包の回数 | 極限点の階数 |
| グラフ | 頂点 | 時間ステップ | 最短経路長 |
| フィルトレーション | 事象 | 時刻 | 停止時間 |
| 証明論 | 命題 | 推論の深さ | 最小証明深さ |

すべてが「**リソースの最小量**」という概念で統一される。

## Bourbakiの精神と現代的拡張

### Bourbakiの母構造

1. **代数的構造**: 群、環、体
2. **順序構造**: 順序集合、束
3. **位相構造**: 位相空間

### 構造塔による統合

構造塔は、これらに**時間軸**を加える：

```
母構造 + 時間発展 = 構造塔
```

### 現代的拡張：モダリティ

Bourbakiの静的な構造に、**動的な視点**を導入：

- 静的: 「集合 A は構造 S を持つ」
- 動的: 「集合 A は時刻 t で構造 S を獲得する」

これにより：
- 時相論理との統合
- 計算理論との接続
- 証明論への応用

が可能になる。

```
         構造塔
           |
    ┌──────┼──────┐
    |      |      |
  代数    幾何    論理
 (生成)  (到達)  (証明)
```
