課題4（部分分数）

証明方針の分岐が多い: 原始関数として F x = log x - log (x+1) を取るのが自然だが、
導関数を「分解形」1/x - 1/(x+1) のまま使うか、
「既約形」1/(x(x+1)) に書き換えて使うかで、必要な書き換えや可積分性の用意が変わる。
領域・非零条件の管理: Set.uIcc 1 2 上で
1 ≤ x（uIcc の左右どちらの枝でも取り出す）、
x ≠ 0, x+1 ≠ 0（log の導関数、inv₀、field_simp で必須）、
を確実に作る必要がある。
FTC の要求を満たす整合性:
intervalIntegral.integral_eq_sub_of_hasDerivAt は
uIcc 上の HasDerivAt、
被積分関数そのものの IntervalIntegrable（関数を導関数と一致させる）を要求。
導関数を分解形で示したなら、積分する関数も分解形で可積分にし、元の被積分関数 1/(x(x+1)) とは
field_simp か
両辺に (x(x+1)) を掛けて手計算で
a.e. 同値を示し、integral_congr_ae で置換するのが安全。
補題名・仮定の差異に注意:
Real.hasDerivAt_log はバージョンにより 0 < x でなく x ≠ 0 を仮定する版がある。
Real.log_div も ≠ 0 の仮定を要求する版がある。
one_div_sub_one_div のような補題名は存在しないため、field_simp や手計算で処理する。
自動化の落とし穴:
逆数・積の simp は再帰が深くなりがち。field_simp に適切な非零仮定を与えるか、両辺に共通因子を掛けて mul_left_cancel₀ で約す方が安定。
uIcc と Icc の食い違いにより型が合わなくなることがある。必要なら uIcc に統一し、Ioc→Icc の変換を明示する。
課題5（面積 ∫₀¹ (x − x²) = 1/6）

コンセプトは易しいが、Lean では小さな技術点がある:
線形性の適用には両項の IntervalIntegrable が必要。連続性から出す。
既知のべき乗の積分は環境により補題名が異なることがある（integral_pow か名前空間付き）。a,b 引数、指数が Nat であること、pow_one などの簡約を適切に入れる必要がある。
integral_pow 2 はしばしば ((1)^(3) - (0)^(3))/3 の形で出るので、simp で 1/3 に落とす段取りを用意する。
被積分関数の整形は by ring/ring_nf で、和と差の整形は by simp（a + -b = a - b）で安定する。
最後の数値は norm_num が手堅い。
まとめ

課題4は、数学的には単純な部分分数だが、Lean では
領域・非零条件・集合（uIcc/Icc）の整合、
導関数の形と積分関数の一致のとり方（分解形か既約形か）、
自動化戦略（field_simp と simp の使い分け）
が絡み合うため難度が上がる。
課題5は本質的には容易だが、補題名・引数・簡約の段取りが噛み合わないと小さなエラーが出やすい。
