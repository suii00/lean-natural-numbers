# A — 実装完成型：`continuous_quotient_map`（位相群の商写像の連続性）

要点（結論）

* 正しく設計すれば**数学的にシンプル**で、mathlib の公理化された商・商位相の普遍性から短く導けます。`QuotientGroup.lift` / `QuotientGroup.mk'` を使い，商位相の普遍性（`Quotient.lift`／`Quotient.lift_mk` 型の補題）を用いて連続性を下ろすのが王道です。([leanprover-community.github.io][1], [GitLab][2])

主要障害点／注意点

* 「quotientToRange φ」がどの写像であるか（`G ⧸ ker φ → φ.range` を `QuotientGroup.lift` で構成）をまず明確にする必要があります（`N ⊆ ker φ` 型の前提整理）。`lift` の well-definedness は代数的計算で消えます。([leanprover-community.github.io][1])
* 連続性: 一般に商位相上の写像の連続性は「商射に合成する形」を使って示します。具体的には `Continuous (quotientToRange φ)` を示すために、`quotient.mk`（商写像）との合成 `quotientToRange φ ∘ quotient.mk = …` を書き、`Continuous.comp` と `continuous_quot_lift` / `Quotient.lift` の連続性補題で降ろします。`Quotient.lift` と関連補題は mathlib にあります。([GitLab][2])

実装スケッチ（Lean / tactics）

```lean
-- 想定インポート
import Mathlib.GroupTheory.QuotientGroup.Defs
import Mathlib.Topology.Algebra.Group.Quotient  -- 位相群の商に関する補題群
open QuotientGroup

-- 1) 定義（代数部）
def quotToRange (φ : G →* H) : (G ⧸ φ.ker) →* φ.range :=
  QuotientGroup.lift (φ.ker) (fun g => ⟨φ g, ⟨g, rfl⟩⟩) <proof_welldefined>

-- 2) 連続性の証明骨子
theorem continuous_quotient_map {G H : Type*} [TopologicalGroup G] [TopologicalGroup H]
    (φ : G →* H) (hφ : Continuous φ) :
  Continuous (quotToRange φ) := by
  -- (a) show quotToRange φ ∘ Quotient.mk = some_continuous_map
  -- (b) use continuity of Quotient.mk and the universal property: continuous_quot_lift
  -- tactics: simp, continuity, apply continuous_quot_lift (or continuity_quotient_lift lemma)
  admit
```

具体的補題（使用候補）: `QuotientGroup.lift`, `Quotient.mk` / `Quotient.lift_mk`、`continuous_quot_lift` 系（Quotient の連続性補題）。([leanprover-community.github.io][1], [GitLab][2])

実装難度：**中**（補題名を正確に呼び出し、商位相に関する小補題を分解して埋める作業が必要）

# B — 高次代数構造型：`five_lemma`（完全列における五項補題）

要点（結論）

* 五項補題はホモロジー代数の標準定理で、mathlib では**アーベル圏/加群の図追跡レマ**として既に整備済みです。category\_theory 側に four/five lemmas のファイルがあり、その API を使えば短く書けます。([leanprover-community.github.io][3])

主要障害点／注意点

* 五項補題を「多重の `Module` 型パラメータで直接手書き」するより、**アーベル圏（Abelian category）としての一般定理**を import して適用するほうが理想です（同一の抽象証明を繰り返す必要がなくなる）。mathlib の `category_theory.abelian.diagram_lemmas.four` などを活用すると良いです。([leanprover-community.github.io][3])
* もし「素のLinearMapで具体的に書く」場合は、exactness（完全性）や射の可逆性・単射性・全射性を usetactics (`exact`, `apply`, `have`, `rw`, `simp`, `diagram_chasing` style) で詰める必要があります。

実装スケッチ（Lean / tactics）

```lean
-- 推奨経路：既存の抽象版をimportして適用
import Mathlib.CategoryTheory.Abelian.DiagramLemmas.Four  -- four/five lemmas 実装済み
open CategoryTheory

-- もし Module レベルで特殊化するなら：
theorem five_lemma_modules {R : Type*} [Ring R] {M₁ ... M₅ : Type*} ... :
  (diagram commutative and rows exact) → Function.Bijective h₃ := by
  -- tactic outline:
  -- 1. Turn exactness hypotheses into monomorphism/epimorphism facts (use `exact` lemmas).
  -- 2. Use the categorical five lemma (specialize the general theorem).
  apply CategoryTheory.Abelian.five_lemma ... -- instantiate arguments
```

使用補題：`LinearMap.quotKerEquivRange` 等も使えます（加群の第一同型）。([leanprover.github.io][4])

実装難度：**低→中**（抽象定理をそのまま呼び出すなら低、手で diagram chase するなら中）

# C — 圏論的統一型：初等トポスの内部言語での同型定理

要点（結論）

* 命題の数学的内容は「任意射 φ を epi ≫ mono に分解する（epi followed by mono）という **像因子化（image factorization）**」に対応します。これは初等トポス（elementary topos）における標準的な性質：像の存在・epi/mono の因子化など。**ただし**，mathlib における「初等トポス（ElementaryTopos）」とその内部言語が**完全に整備されているわけではない**ので、まずはトポス構造（有限極限、部分オブジェクト分類子、指数対象）を定義してから進める必要があります。mathlib の全体目次にトポス理論が網羅的にあるわけではない点に注意。([leanprover-community.github.io][5])

主要障害点／注意点

* mathlib には **圏論・有限極限・画像因子化** の基礎はあるものの（CategoryTheory API）、初等トポス全体をすっぽり実装済みのパッケージは限定的です。従って「`ElementaryTopos` のクラス定義 → 内部オブジェクト言語 → 全射⇔単射の因子化を内部で証明する」は**大きな作業**になります。
* 実用的戦略は次のどちらか：

  1. **外部的に像因子化を使う方法**（圏論的 image/coequalizer を使って `φ = π ≫ ι` を構成し、`π` が epi，`ι` が mono を示す）— これは CategoryTheory の `Image` / `Factorization` API を利用するアプローチ。
  2. **内部言語で完全に記述**する方法（内部の `Subobject`、`Classifying` を使って内部的証明を書く）— よりエレガントだが重い。

実装スケッチ（外部的 image を使う路線）

```lean
-- 依存: CategoryTheory image/factorization API
import Mathlib.CategoryTheory.Image -- image, image.ι, image.ι_is_mono など

-- E : Category with finite limits and subobject classifier (i.e. topos axioms)
class ElementaryTopos (E : Type*) [Category E] :=
  (has_finite_limits : ...)
  (has_subobject_classifier : ...)

theorem topos_internal_isomorphism {E : Type*} [Category E] [Elem : ElementaryTopos E]
  {G H : E} (φ : G ⟶ H) :
  ∃ (Q : E) (π : G ⟶ Q) (ι : Q ⟶ H),
    IsEpimorphism π ∧ IsMonomorphism ι ∧ φ = π ≫ ι := by
  -- Proof outline:
  -- 1. take Q := Image φ (category_theory.image φ)
  -- 2. set π := image.ι.hom? (or appropriate projection)
  -- 3. use `image.ι_is_mono` and `image.factors` lemmas to get φ = π ≫ ι
  admit
```

関連 API：`CategoryTheory.Image` / `Image.ι_is_mono` / `image.fac` 等（mathlib の CategoryTheory にある）。（実装時は正確な名前を import して使う）([GitHub][6])

実装難度：**高**（初等トポスの完全な API を整えるか、CategoryTheory 側の image/factorization を適切に引き出す必要があるため）
