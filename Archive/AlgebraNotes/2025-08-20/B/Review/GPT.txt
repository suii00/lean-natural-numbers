# BourbakiTopologicalQuotientMap（位相群の商写像）

**構造概要図**

* ファイルは「商位相の概念」「連続商写像定理」「位相的図式追跡」などのセクションに分かれており、商位相に関する占位的構造 `QuotientTopologyConcept` を定義しています（概念定義部分）。

**論理的依存関係**

* 主要宣言は商位相の普遍性→連続性→位相群同型へと流れる設計（`QuotientTopologyConcept` → `continuous_quotient_map_basic` → `topological_group_isomorphism_continuity`）。ただし現在はすべて *占位（True/trivial）* で、具体的補題への依存は明示されていません。

**未完成部分の戦略的分析**

* 未完成の実務的リスト：

  1. 商位相（`G ⧸ ker`）の **正確な topological instance** の import と `QuotientGroup.instTopologicalSpace` 等の利用がない（占位）。
  2. 連続性証明は currently `trivial`／占位：`Continuous` の降ろし方（`Quotient.lift` の連続性補題）を入れる必要あり。
  3. 開写像・像の位相的性質に関する補題群（`IsOpenMap` の伝播）を分割補題化する必要あり（`IsOpenMap φ` → `IsOpenMap ψ`）。

**優先度付け**

* 最重要：`quotientToRange` の algebraic well-definedness（既に実装済みのことが多い）→商位相インスタンスの import→`continuous_quot_lift` 的補題で連続性を降ろす（中難度）。

**具体的最適化コード例（差し替え提案）**

* 使うべき import と連続性証明の骨格（mathlib4 を想定）：

```lean
import Mathlib.GroupTheory.QuotientGroup.Basic
import Mathlib.Topology.Algebra.Group.Quotient  -- quotient の位相補題群
open QuotientGroup

def quotToRange (φ : G →* H) : (G ⧸ φ.ker) →* φ.range :=
  QuotientGroup.lift (φ.ker) (fun g => ⟨φ g, ⟨g, rfl⟩⟩) (by
    intros g h hmem; simp [MonoidHom.mem_ker] at hmem;
    have : φ (g⁻¹ * h) = 1 := hmem; simp [map_mul, map_inv] at this;
    exact inv_mul_eq_one.mp this)

theorem continuous_quotient_map {G H : Type*} [TopologicalGroup G] [TopologicalGroup H]
    (φ : G →* H) (hφ : Continuous φ) :
  Continuous (quotToRange φ) := by
  -- (1) define u : G → φ.range by u g = ⟨φ g, _⟩ (continuous by hφ and subtype topology)
  -- (2) use `Quotient.lift` / `continuous_quot_lift` style lemma: if u is continuous and constant on ker
  --     cosets then the induced map on quotient is continuous.
  -- tactics: refine continuous_quot_lift ?; exact ...
  admit
```

→ `admit` 部分は `continuous_quot_lift` / `continuous_quotient_mk` 補題を呼べば消せます。

# Bourbaki.HomologicalAlgebra（五項補題）

**構造概要図**

* `FiveLemmaDiagramConcept` 等で図式と「完全性」を占位的に定義し、図式追跡（diagram chasing）セクションを持つ。実体は概念的スタブで占められています。

**論理的依存関係**

* 目標は「完全列の条件→h₃ の同型」だが、現状は抽象テンプレート（概念的補題）しかなく、mathlib の *category\_theory/abelian* 側の既存定理を結びつけていない。

**未完成部分の戦略的分析**

* 2つの実装方針：

  1. **具体版（Module/LinearMap）**：`LinearMap` と `Module` を直接扱い、exactness を `Kernel`, `Range` の補題で展開し手で diagram chase する（手間はあるが理解しやすい）。
  2. **抽象版（圏論）**：`CategoryTheory.Abelian.five_lemma` 等、mathlib の既存抽象定理を *Module* カテゴリに適用する（簡潔で推奨）。

**優先度付け**

* 低〜中難度。既存の `abelian` 側定理を使うなら短時間で完了。図式追跡を手で書くなら少し長め。

**具体的最適化コード例（抽象→具現化）**

```lean
import Mathlib.CategoryTheory.Abelian.DiagramLemmas.Four  -- 例：four/five lemmas

theorem five_lemma_modules {R : Type*} [Ring R]
  {A B C D E : ModuleCat R} {A' B' C' D' E' : ModuleCat R}
  (comm : -- diagram + exactness hypotheses ...) :
  IsIso (homOfC C C') := by
  -- apply CategoryTheory.Abelian.five_lemma / instantiate hypotheses
  admit
```

→ `admit` は具体的な exactness 条件を `Abelian.five_lemma` に合わせて与えれば消えます。

# Bourbaki.ToposTheory（初等トポスの内部同型）

**構造概要図**

* `ElementaryToposConcept` / `ImageFactorizationConcept` 等を用いてトポス概念を「概念的に」モデル化しているが、内部はほとんど `True / trivial` のスタブになっています（＝実際の構成・証明は未着手）。

**論理的依存関係**

* 立て方は「初等トポスの公理」→「像因子化」→「内部言語での因子化」の流れで正しい。しかし mathlib の CategoryTheory API（`Image` / `Image.ι_is_mono` / `image.fac` 等）を直接利用していないため実装が空白です。

**未完成部分の戦略的分析**

* 最大の課題：

  1. **`ElementaryTopos` の concrete class 化**：Finite limits / subobject classifier / exponentials を具備するクラスとして定義して、CategoryTheory API と接続する。
  2. **外部的 image/factorization を使う実装**：内部言語で全部やるよりまず `Image φ`（圏論側）を使って `φ = π ≫ ι` を与え，`π` を epi，`ι` を mono として示すのが現実的。

**優先度付け**

* 高難度・高価値。研究的には最もインパクトがあるが実装工数が大きい（CategoryTheory の深い整備と topos 独自の補題が必要）。

**具体的最適化コード例（実用路線）**

```lean
import Mathlib.CategoryTheory.Limits.Shapes.Image
import Mathlib.CategoryTheory.Category.Basic

class ElementaryTopos (E : Type*) [Category E] :=
  (has_finite_limits : HasFiniteLimits E)
  (has_subobject_classifier : HasSubobjectClassifier E)
  -- (exponentials etc.)

theorem topos_internal_isomorphism {E : Type*} [Category E] [ET : ElementaryTopos E]
  {G H : E} (φ : G ⟶ H) :
  ∃ (Q : E) (π : G ⟶ Q) (ι : Q ⟶ H),
    IsEpimorphism π ∧ IsMonomorphism ι ∧ φ = π ≫ ι := by
  let img := image φ
  use img.obj, image.ι φ, image.factorisation φ
  -- image.ι_is_mono and image.fac provide the rest
  split; try exact image.ι_is_mono φ; -- etc.
  admit
```

→ `admit` 部分は `image` API の正確な名前で置換すれば消えます（関数名は mathlib バージョンで変わりうるので調整が必要）。
