# S7_CH 実装が難しい理由

## 1. 局所化の元は明示的な「分数」ではない
Mathlib の `Localization S` は抽象的な商構成であり、Lean 上では `HDiv`（除算）インスタンスが定義されていません。そのため形の上では `f 6 / f 4` を書けず、目標の等式

```lean
f 6 / f 4 = f 3 / f 2
```

をそのまま扱うことができません。`IsLocalization.mk'` を使えば分数を表現できますが、記法が大きく変わり、補題の適用も直接的ではなくなります。

## 2. `IsLocalization.mk'` と等式判定補題の扱いの難しさ
局所化の元を `IsLocalization.mk' R n ⟨m, hm⟩` で表現すると、等式判定には `IsLocalization.mk'_eq_iff_eq` のような補題を使う必要があります。しかしこの補題は「クロス乗算」に対応する式を `algebraMap` を通して示す形式で与えられており、既存の目標をその形に変形するには追加の補題や計算が必要です。特に `IsLocalization.mk'` から Lean の除算表現へ変換する補助補題が標準では提供されていないため、自前で証明を積み上げる必要があります。

## 3. 適切な部分モノイドの選択とその影響
課題文では $S=\{1,2,4,8,\dots\}$ を想定していますが、Lean 側でこの集合を `Submonoid.closure {2}` として構成すると $3$ や $5$ が自動的には含まれません。その結果 `f 6`, `f 3`, `f 5` のような項を扱うには、$3$ や $5$ もモノイドに含まれるよう生成集合を拡張するか、`IsLocalization` の射像が単元になることを別途示す必要があります。生成集合を増やせば目的の等式は扱いやすくなる一方で、課題文との整合をどう保つかという設計判断が求められます。

## 4. 代替方針の検討が必要
- **目標の書き換え**: 等式を `IsLocalization.mk'` の形に書き換えて証明する方針に変更する。
- **モノイドの再定義**: $S$ を全ての非零整数の集合に拡大し、局所化を $\mathbb{Q}$ と同一視できるようにする。
- **補助補題の追加**: `IsLocalization.mk'` と除算記法の往来を扱う補題を自作する、あるいは `IsLocalization.eq` 系の補題を活用してクロス乗算の等式を直接示す。

これらの調整なしに `f 6 / f 4 = f 3 / f 2` を Lean で証明するのは難しく、CH は基礎問題より一段高い抽象度と実装コストを要求します。
