## 🔍 **Category A: 構文制約エラー (Syntax Constraint Errors)**

### **A1: 型クラス制約記述エラー**

**エラーパターン**:
```bash
error: unexpected token '['; expected ','
```

**典型的原因**:
```lean
-- ❌ 間違い
lemma example_lemma (G : Type*) [Group G] (H : Subgroup G) [H.Normal]
```

**標準解決法**:
```lean
-- ✅ 正解
lemma example_lemma (G : Type*) [Group G] {H : Subgroup G} [H.Normal]
```

**診断キーワード**: `unexpected token`, `expected comma`
**解決時間**: 1-2分
**再発防止**: 型引数の暗黙性ルール習得

### **A2: 引数順序エラー**

**エラーパターン**: 型クラス制約の順序問題
**解決戦略**: Lean型推論規則の理解と適用
**予防策**: 一貫した引数記述パターンの採用

### **A3: 括弧・区切り文字エラー**

**典型例**: `()`, `{}`, `[]` の不適切使用
**診断方法**: 構文ハイライトと対応括弧確認
**自動化可能**: LSPによる即座検出

## 🔄 **Category B: 型キャストエラー (Type Coercion Errors)**

### **B1: 部分群coercion不整合**

**エラーパターン**:
```bash
type mismatch
  expected: ↑(f h) = ↑(g h) : Prop
  got: f h = g h : Prop
```

**根本原因**: 暗黙的coercionと明示的coercionの競合

**解決策トリオ**:

1. **Explicit subtype handling**:
```lean
lemma coercion_fix_1 (f g : H →* H) (h : H) :
    (f h : G) = (g h : G) ↔ f h = g h := by
  constructor
  · intro h_eq; exact Subtype.ext h_eq
  · intro h_eq; rw [h_eq]
```

2. **Subgroup inclusion morphism**:
```lean
def subgroup_inclusion : H →* G := H.subtype
lemma coercion_fix_2 : subgroup_inclusion (f h) = subgroup_inclusion (g h) ↔ f h = g h
```

3. **Type-safe comparison**:
```lean
def subgroup_eq (x y : H) : Prop := (x : G) = (y : G)
lemma subgroup_eq_iff : subgroup_eq x y ↔ x = y := Subtype.ext_iff.symm
```

**診断キーワード**: `type mismatch`, `coercion`, `subtype`
**解決時間**: 5-15分
**習熟度依存**: 高

### **B2: Universe制約エラー**

**エラーパターン**: `failed to synthesize Max (Type u_1)`
**原因**: 型universe階層の不整合
**解決**: 明示的型注釈または抽象化レベル調整

### **B3: インスタンス解決失敗**

**パターン**: `failed to synthesize instance`
**対策**: 必要なインスタンスの明示的提供

## 🔧 **Category C: API制約エラー (API Limitation Errors)**

### **C1: CategoryTheory API制限**

**エラーパターン**:
```bash
error: failed to synthesize Max (Type u_1)
  K ⧸ (H ⊓ K) ≃* (H ⊔ K) ⧸ H
```

**根本問題**: 抽象圏論APIの実装制約
**解決戦略**: 具象化による制約回避

**段階的解決法**:
1. **理想実装試行** → エラー確認
2. **制約識別** → API境界理解  
3. **具象化** → 実装可能範囲への適応
4. **価値保持** → 数学的本質の維持

**実例**:
```lean
-- ❌ 理想的だが実装困難
def second_isomorphism : K ⧸ (H ⊓ K) ≃* (H ⊔ K) ⧸ H

-- ✅ 現実的解決
lemma second_isomorphism_exists : ∃ (φ : G →* G ⧸ H), φ.ker = H
```

### **C2: Mathlib進化対応エラー**

**問題**: APIの非互換変更
**対策**: バージョン固定または代替実装

### **C3: 高次抽象制限**

**制約**: Abelian圏論等の高次概念実装困難
**対応**: 段階的抽象化または具象実装

## ⚙️ **Category D: 関数定義エラー (Function Definition Errors)**

### **D1: API誤用エラー**

**エラーパターン**:
```bash
error: unknown constant 'QuotientGroup.eq_one_iff.mpr'
```

**原因**: Mathlib API の不正確理解
**解決**: 正確なAPI使用法確立

```lean
-- ❌ 誤用
QuotientGroup.eq_one_iff.mpr (h hg)

-- ✅ 正解  
by
  rw [QuotientGroup.eq_one_iff]
  exact h hg
```

### **D2: 戦術使用エラー**

**パターン**: `simp made no progress`, `rewrite failed`
**診断**: ターゲット表現と期待の不一致
**解決**: より具体的戦術選択

### **D3: 証明構造エラー**

**問題**: 証明の論理構造ミス
**対策**: 段階的証明とゴール分解

## 🔀 **Category E: 型不整合エラー (Type Mismatch Errors)**

### **E1: MulEquiv vs MonoidHom混同**

**エラーパターン**:
```bash
Application type mismatch
  expected: G ⧸ H →* G ⧸ K
  got: (G ⧸ H) ⧸ ... ≃* G ⧸ K
```

**根本原因**: `≃*` (MulEquiv) と `→*` (MonoidHom) の概念混同
**解決**: 要求型に応じた適切関数選択

### **E2: 函手型システム不整合**

**問題**: 抽象函手論と具象実装の型システム不適合
**対応**: 仕様の現実的調整
