# S12_P02/P03 実装時のエラーメモ

## 1. `Tactic "assumption" failed`（`padicNorm 5 25`）
- 発生コマンド: `lake build MyProjects.IUT1.S12`
- 主なメッセージ:
  - `Tactic 'assumption' failed`
  - `⊢ (5 ^ padicValRat 5 5)⁻¹ * (5 ^ padicValRat 5 5)⁻¹ = 5⁻¹ * 5⁻¹`
- 原因: `padicNorm.mul` の出力が `padicValRat` を含む形の等式として残り、ゴールの右辺 `5⁻¹ * 5⁻¹` と一致しなかった。
- 対処: `calc` ブロック内で `simpa [h25, hnorm5] using padicNorm.mul …` を用い、両辺を `simp only [hnorm5]` で簡約可能な形に整えてから `norm_num` へ渡した。

## 2. `Type mismatch … padicNorm.mul`（`padicNorm 5 (25 : ℚ)`）
- 発生コマンド: `lake build MyProjects.IUT1.S12`
- 主なメッセージ:
  - `Type mismatch: After simplification, term padicNorm.mul 5 5 …`
- 原因: `padicNorm.mul` の引数名を `a`/`b` と誤って指定し、Lean が意図したメソッドシグネチャと一致せずに別種の補題を展開してしまったため。
- 対処: 引数名をデフォルトの `q`/`r` に戻し、`padicNorm.mul (p := 5) (q := (5 : ℚ)) (r := (5 : ℚ))` の形に修正した。

## 3. `Type mismatch … padicValNat` / `Unknown identifier Fact`
- 発生コマンド: `lake build MyProjects.IUT1.S12`（途中の一時編集時）
- 主なメッセージ:
  - `Function expected at padicValNat …`
  - `Unknown identifier Fact`
- 原因: 一時的に `ℚ` や `Fact` などの型を書き換える際にプレースホルダ `?` を残した状態で保存してしまい、Lean が型推論できなくなった。
- 対処: 影響範囲を洗い出し、明示的に `ℚ`・`ℤ`・`Fact (Nat.Prime p)` を書き直した上で、`simp` 系の補題を正しく参照するよう全体を再編集した。
